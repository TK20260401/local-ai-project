## 🛠️ タスク2：日本語の文章構造を壊さない「再帰的チャンク分割」の自作

### 1. 目的

機械的に文字数（例：300文字）でスパッと区切る手法では、重要な一文が途中で引き裂かれ（例：「翌々月の精算となり」で終了）、意味の連続性が失われる。
日本語の句読点（。、\n）のキリの良い境界線をシステムが自動で検知し、意味の塊を壊さずに分割する。

### 2. 効果

* **リランクスコアの安定化**: 意味が1つのチャンク内に綺麗に閉じ込めるため、CrossEncoderが文脈を正しく理解し、適切なスコアを付与できる。
* **LLMの理解度向上**: 途切れた文章ではなく、1つの「完成された段落」としてインプットされるため、ハルシネーション（嘘）が劇的に減る。

### 3. コアロジック（コード解説）

まずは「改行」で分割を試み、それでも文字数が溢れる場合は「句点（。）」で区切る、という処理を再帰的に繰り返す。

```python
def recursive_split_japanese(text, max_chars=300):
    \"\"\"日本語の句読点を最優先してキリよく分割するプロ仕様ロジック\"\"\"
    if len(text) <= max_chars:
        return [text]
    
    chunks = []
    # 1. 優先度の高い区切り文字でまずバラす
    for paragraph in text.split("\\n"):
        if len(paragraph) <= max_chars:
            chunks.append(paragraph)
        else:
            # 2. 改行でも収まらない場合は「。 」で再分割
            sentences = paragraph.split("。")
            current_chunk = ""
            for s in sentences:
                if not s.strip(): continue
                s_with_period = s + "。"
                if len(current_chunk) + len(s_with_period) <= max_chars:
                    current_chunk += s_with_period
                else:
                    if current_chunk: chunks.append(current_chunk)
                    current_chunk = s_with_period
            if current_chunk: chunks.append(current_chunk)
    return chunks

```
